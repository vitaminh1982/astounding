3 — Usage Rules

Do / Don't — applies to both themes unless noted.

3.1 · Dropdown Select

✓ Do ✗ Don't
Use rgba(255,255,255,0.05) (dark) or rgba(0,0,0,0.10) (light) for the default trigger border. These match the --border / --border-strong tokens. Hardcode a solid gray border — it will look wrong on the glass sidebar in both themes.
Show a ✓ checkmark on the selected item. Make the row background rgba(255,255,255,0.05) (dark) or rgba(0,0,0,0.05) (light). Rely on bold font alone to indicate selection — it reduces clarity for colorblind users.
Always include the green "+ Create new project" CTA link at the panel bottom using the themed primary green. Use a full-width solid button inside the panel — it visually competes with the main CTA in the sidebar.

3.2 · Popup Menu

✓ Do ✗ Don't
Use --popover as the panel background: #0f1930 (dark) or #ffffff (light) — distinct from the sidebar glass. Use the sidebar glass background for the popup — it merges visually into the sidebar and loses depth.
Menu items: 12px / 500-weight text. Default: #a3aac4 (dark) or #374151 (light). Hover: #dee5ff / #111827. Use bright white text at rest in dark mode — it creates no hierarchy between default and active states.
Separate sections (User · Workspace · Settings · Actions) with 1px dividers using rgba(255,255,255,0.05) / rgba(0,0,0,0.07). Use opaque dividers or padding-only gaps — the subtle transparency keeps the glass aesthetic intact.

3.3 · Work / Studio / Govern Switch

✓ Do ✗ Don't
Active pill: rgba(255,255,255,0.15) (dark) or #ffffff (light), always with a subtle shadow. The floating pill is the Sendplex IA navigation signature. Use a colored (green/brand) background for the active pill — it's too loud for a mode selector.
Container track: rgba(255,255,255,0.05) / rgba(0,0,0,0.05) + matching border. Keep the track visible but subtle. Remove the container track — the three segments need a visual anchor to read as a group.
Animate the active pill indicator with Framer Motion layoutId + spring(380, 30) for a fluid slide between segments. Use CSS transitions alone for the indicator — instant snap or simple fade breaks the physicality of the component.

3.4 · CTA Button

✓ Do ✗ Don't
Both modes: use #22c55e → #34d399 gradient. Text: dark green (#0d2b1a) in dark mode, white (#ffffff) in light mode. Use different gradients or hardcoded text colors outside of the design system specs.
Show this button only when workspace.projects.length === 0. Replace with the dropdown trigger once projects exist. Show both CTA and dropdown simultaneously — they represent mutually exclusive workspace states.
Text: 12px / 600-weight / font-family: var(--font-body) with a + icon at 13px. Radius: --radius-xl. Use #ffffff or #000000 hardcoded as text color — always reference --color-on-primary or the themed token.
