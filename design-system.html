## Semantic and accessibility rules

### Element rules (non-negotiable)
- A demo of a button → <button type="button">, never <div role="button">
- A demo of a heading → correct <h1>–<h6> tag with aria-hidden="true" to avoid polluting document outline
- A demo of a list → <ul>/<li>, never <div> stacks
- A divider → <hr> with CSS border:none (no role="separator" needed)
- A progress bar → <div role="progressbar" aria-valuenow aria-valuemin aria-valuemax>
- Decorative icons, chevrons, emojis → aria-hidden="true"
- Functional icons → role="img" aria-label="[description]"

### Style rules (non-negotiable)
- Zero inline style="" on structural or typographic properties
- All visual properties go in named CSS classes in the <style> block
- Only acceptable inline style: color swatches where the value IS the content (e.g. style="background:#4ade80")
- Width: components use width: 100% — the .state-demo wrapper (244px) provides the constraint
- All colors, spacing, radius and shadow values must reference CSS custom properties (--t-* tokens)
- Never hardcode hex values in component styles — always use the token

### Naming convention
- CSS classes follow BEM: .c-[component]__[element]--[modifier]
- State modifiers: --hover, --active, --focus, --disabled, --open, --selected
- New token fill/width classes: .c-[component]__[element]--[value]

### React notes to include as HTML comments
- Dynamic values (widths, counts, labels) → add <!-- React: [explanation] --> comment
- Interaction states → add comment mapping CSS modifier to React prop or CSS pseudo-class
- Conditional rendering → add comment explaining the condition

3 — Usage Rules

Do / Don't — applies to both themes unless otherwise noted.

3.1 · Dropdown Select

Dropdown Select — usage rules
✓ Do ✗ Don't
Use --t-border-strong for the default trigger border. This token resolves correctly in both themes automatically. Hardcode a border color — it will look wrong against the glass sidebar background in both themes.
Show a ✓ checkmark on the selected item with --t-item-hover-bg as the row background. Rely on font weight alone to indicate selection — insufficient for accessibility.
Always include the "+ Create new project" link at the bottom of the panel, colored with --t-primary. Use a full-width solid button inside the panel — it visually competes with the main sidebar CTA.

3.2 · Popup Menu

Popup Menu — usage rules
✓ Do ✗ Don't
Use --t-popover as the panel background — it visually separates from the sidebar glass and creates depth. Use the sidebar glass background for the popup — it merges into the sidebar and loses layering.
Item text: 12px / 500. Default: --t-muted-fg. On hover: --t-foreground. Use pure white text at rest in dark mode — no hierarchy between default and active states.
Separate sections with --t-border (1px). The token ensures the correct transparency in both themes. Use opaque dividers or padding alone — it breaks the glass aesthetic.

3.3 · Work / Studio / Govern Switch

Mode Switch — usage rules
✓ Do ✗ Don't
Active pill: --t-switch-active-bg + --t-switch-active-shadow. The floating pill is Sendplex's AI navigation signature. Use a colored (green/brand) background for the active pill — too loud for a mode selector.
Track container: --t-switch-bg + --t-switch-border. The track stays visible but subtle. Remove the track container — the three segments need a visual anchor to read as a group.
Animate the active pill indicator with Framer Motion layoutId + spring(380, 30). Use CSS transitions alone for the indicator — instant snap or simple fade breaks the physicality of the component.

3.4 · CTA Button

CTA Button — usage rules
✓ Do ✗ Don't
Same gradient in both themes: #22c55e → #34d399. Text: --t-cta-text (resolves to dark green or white depending on the theme). Hardcode #000000 or #ffffff as the text color — always reference --t-cta-text.
Show this button only when workspace.projects.length === 0. Replace it with the dropdown as soon as a project exists. Display CTA and dropdown simultaneously — they represent mutually exclusive workspace states.
Text: 12px / 600 / --font-body with a + icon. Radius: --radius-xl. Adjust the radius or typography to make the button "stand out more" — hierarchy comes from the gradient, not the size.
